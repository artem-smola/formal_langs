# Китайский почтальон — решение на C++

Решается классическая задача китайского почтальона для связного
неориентированного взвешенного графа:
найти минимальную длину замкнутого маршрута, проходящего по каждому ребру хотя бы один раз.

## Формат ввода

Из `stdin`:

1. `n m` — число вершин и рёбер.
2. `m` строк: `u v w` — ребро между `u` и `v` длины `w`.

Ограничения:
- граф неориентированный;
- веса `w >= 0`;
- все вершины с ненулевой степенью лежат в одной компоненте связности.

Ограничение текущей реализации:
- число вершин нечётной степени `k` должно быть `<= 22`
  (из-за точного DP по битмаске).

## Формат вывода

- сумма длин всех рёбер;
- добавочная длина (стоимость дублирования рёбер);
- минимальная длина маршрута;
- попаривание вершин нечётной степени.

## Используемый алгоритм

1. Считается сумма длин всех рёбер.
2. Находятся вершины нечётной степени.
3. Для них считаются кратчайшие пути (Дейкстра от каждой нечётной вершины).
4. Ищется минимальное попаривание нечётных вершин DP по битмаске.
5. Ответ: `sum_edges + min_matching_cost`.

## Сборка

```bash
make
```

## Запуск

```bash
./postman < input.txt
```

или

```bash
make run < input.txt
```

## Очистка

```bash
make clean
```
