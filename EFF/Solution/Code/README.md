# EFF\_k(alpha) — решение на C++

Программа строит множество `EFF_k(alpha)` для КС-грамматики:

- рассматриваются только правосторонние выводы;
- для каждой полученной терминальной цепочки берется префикс длины `k`
  (или вся цепочка, если ее длина меньше `k`);
- в `EFF` попадают только те листья вывода, где последнее примененное правило
  не является `epsilon`-продукцией.

Дополнительно программа печатает `ALL_PREFIXES` — те же префиксы без
фильтра по последнему правилу.

## Формат ввода

Программа поддерживает два формата.

### 1) Legacy (краткий)

Вариант A (совместимость со старыми тестами):

1. `k`
2. `p`
3. `p` строк продукций
4. одна или несколько строк `alpha`

Вариант B:

1. `k` или `k <число>`
2. далее в произвольном порядке:
   - продукции `A -> ...`
   - строки `alpha`
   - `max_nodes <число>`
   - `max_form_chars <число>`

### 2) Extended (строгий)

Ввод включает директивы:

- `k <число>`
- `start <НТ>`
- `nonterminals A B C ...`
- `terminals a b c ...`
- одна или несколько строк `alpha ...`
- продукции `A -> ...`
- опционально `max_nodes`, `max_form_chars`

В extended-формате проводится строгая проверка:
- символы в продукциях и alpha должны принадлежать объявленным алфавитам;
- `start` обязан входить в `nonterminals`.

## Формат вывода

Для одного `alpha`:

```text
EFF:
...
ALL_PREFIXES:
...
```

Для нескольких `alpha`:

```text
ALPHA i: ...
EFF:
...
ALL_PREFIXES:
...
```

Если множество пустое, печатается `∅`.
Если сработали лимиты `max_nodes` или `max_form_chars`, дополнительно печатается предупреждение.

## Сборка

```bash
make
```

## Запуск

```bash
./eff < input.txt
```

или

```bash
make run < input.txt
```

## Очистка

```bash
make clean
```
